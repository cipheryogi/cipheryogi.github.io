ChangeMan Promotional Edits

Category: Serena ChangeMan

Overview

ChangeMan's promotional edits feature automatically modifies environmental components (JCL, PROC, BIND cards) when code is promoted across different environments (DEV, QA, SIT, UAT, PROD).

Key Concept

The code always remains in production standards in the ChangeMan package, while parameters adjust automatically for each target environment.

Example

A production job card:

//P4BX5550 JOB (P0BXX,PROD),'PRODUCTION JOB'

Automatically transforms when promoted to development:

//D4BX5550 JOB (D0BXX,DEV),'DEV JOB'

And similarly for QA with Q-prefixed standards.

Implementation

The solution uses FILE-AID within ChangeMan skeleton files, performing string replacements across specified dataset ranges. For example, replacing database references like "DSNPENV" with "DB2TESTS" and library indicators from production to test configurations.

Primary Benefit

Eliminates repetitive manual editing, since the same validated production code automatically adapts to environment-specific requirements during the promotion workflow.
