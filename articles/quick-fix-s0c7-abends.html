Quick Fix for S0C7 Abends

Category: JCL and z/OS Utilities

Overview

A practical JCL solution using FILEAID to remove problematic data causing S0C7 abends (data exception errors).

JCL Solution

//VNDJ29Y JOB (D0B9D,DEV),'VNDJ29',CLASS=A,
//         MSGCLASS=T,NOTIFY=&SYSUID,TIME=(,20)
//STEP01 EXEC PGM=FILEAID
//SYSUDUMP DD SYSOUT=*
//SYSPRINT DD SYSOUT=*
//SYSLIST  DD SYSOUT=*
//SYSTOTAL DD SYSOUT=*
//DD01     DD DSN=VNDJ29.FLATFILE,DISP=SHR
//DD01O    DD DSN=VNDJ29.FLATFILE.NEW,
//            DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(10,10),RLSE),
//            DCB=(VNDJ29.FLATFILE)
//SYSIN DD *
$$DD01 DROP IF=(1,5,NEP)
/*

FILEAID Filter Parameters

ParameterMeaning
NEPNot Equal to Pack format
EQPEqual to Pack format
EQNEqual to Numeric format
NENNot Equal to Numeric format

These parameters allow filtering records based on data format validation at specified positions and lengths within dataset records.
